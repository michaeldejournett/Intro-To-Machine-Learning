\title{Classifying Uber and Lyft Rides into Price Quintiles Using Weather-Enhanced Features}

\author{\IEEEauthorblockN{Ermias Wolde}\IEEEauthorblockA{CSCE 478 --- ML Project}}

\maketitle

\begin{abstract}
We use the Kaggle \emph{Uber \& Lyft Cab prices} dataset (Boston-area rides). Instead of predicting exact dollars, we predict five \emph{price quintile} classes (0--4) so we can use confusion matrices like the project requires. Weather is merged to each ride by pickup neighborhood and closest timestamp. Two PyTorch MLPs are compared: a smaller two-layer net and a deeper one with batch normalization. We report test accuracy and macro-F1 with bootstrap $95\%$ CIs and row-normalized confusion matrices. Sample test accuracies from our run: \ShallowMLPAcc{} (shallow) vs.\ \DeepBatchNormMLPAcc{} (deep); the table lists CIs.
\end{abstract}

\begin{IEEEkeywords}
Classification, neural network, Uber, Lyft, weather data, confusion matrix.
\end{IEEEkeywords}

\section{Introduction}
Rideshare pricing depends on distance, surge, product type, and other factors; weather might matter too. The raw dataset gives price in dollars, but the assignment also calls for confusion matrices, so we split \texttt{price} into five quantile bins (quintiles) and treat the task as 5-way classification. We compare two MLP architectures and evaluate them with accuracy, macro-F1, bootstrap intervals, and normalized confusion matrices.

\section{Problem Formulation}
\textbf{Input:} For each ride we use numeric fields \texttt{distance}, \texttt{surge\_multiplier}, and weather fields \texttt{temp}, \texttt{clouds}, \texttt{pressure}, \texttt{rain}, \texttt{humidity}, \texttt{wind} after merging. Categorical fields \texttt{cab\_type}, \texttt{destination}, \texttt{location} (pickup), \texttt{name}, and \texttt{product\_id} are one-hot encoded. Missing \texttt{rain} in the weather file was set to zero so we did not drop most of the rows.

\textbf{Output:} Class label $y \in \{0,\ldots,4\}$ from \texttt{pd.qcut} on \texttt{price} (five quantile bins).

\textbf{Goal:} Minimize cross-entropy on the training set; we use a validation split and early stopping.

\section{Models}
\begin{itemize}
  \item \textbf{ShallowMLP:} Two hidden layers (128 then 64 units), ReLU, dropout, then softmax via cross-entropy loss.
  \item \textbf{DeepBatchNormMLP:} Four blocks with width $256 \rightarrow 128 \rightarrow 64 \rightarrow 32$, each with batch norm, ReLU, and dropout.
\end{itemize}
Both use AdamW and the same stratified train/validation/test split (see code for sizes).

\section{Metrics and Which Model We Prefer}
We report \textbf{accuracy}, \textbf{macro-F1} (unweighted mean of F1 over the five classes), and \textbf{row-normalized} confusion matrices (each row of counts is divided by its row sum so we see where each true bin goes).

\section{Pros and Cons}
\textbf{What worked:} Merging weather adds signal beyond distance alone; bootstrap CIs give a sense of how stable the metrics are; normalized confusion matrices show which price bins get confused.

\textbf{Limitations:} Quintile labels depend on the whole price distribution, so bins are not fixed dollar ranges; results are only as good as the merge (nearest weather time per neighborhood); we only tried two MLP designs; full-data training on CPU was slow.
