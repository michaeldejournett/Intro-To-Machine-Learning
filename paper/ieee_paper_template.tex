\begin{abstract}
This report studies ride fare prediction for Uber and Lyft trips using supervised learning.
Given 14 trip-level features including distance, time, location, vehicle type, and weather conditions, the goal is to predict fare amount.
We evaluate baseline Linear Regression and ensemble Random Forest models, report test-set performance with 95\% confidence intervals, and compare model
quality using regression metrics and normalized confusion matrices.
The best model, Random Forest, achieves MAE = \textbf{\$1.30}, RMSE = \textbf{\$2.32}, and $R^2=\textbf{0.939}$
with a 95\% confidence interval for MAE of \textbf{[1.238, 1.380]}.
Results indicate that Random Forest provides the best trade-off between accuracy,
stability, and practical usability for this dataset, outperforming the linear baseline.
\end{abstract}

\section{Introduction}
Fare prediction is useful for pricing transparency, dispatch planning, and customer experience.
This project compares linear baseline and ensemble regression models on Uber/Lyft trip data to estimate fares.
Our final results are MAE = \textbf{\$1.30}, RMSE = \textbf{\$2.32}, and $R^2=\textbf{0.939}$,
with 95\% confidence intervals computed from split-level empirical percentiles over repeated holdout runs.
Among the tested methods, Random Forest performs best for this dataset and is selected as the final approach.
The report concludes with model strengths/limitations and practical implications for real-world deployment.

\section{Problem Formulation}
The task is to predict ride fare price in USD from 14 trip and context features.
Inputs include distance in miles, time features such as hour, day of week, and month, pickup and drop-off locations,
vehicle type information such as cab type and service name, and weather conditions such as temperature, humidity, wind, pressure, rain, and clouds.
The target is the trip fare price.
Data was evaluated with repeated 80\%/20\% train-test holdout splits across 20 runs.
Preprocessing used train-split median imputation for missing numeric values and split-aware categorical encoding.
Feature scaling was used for Linear Regression but not for Random Forest.

\section{Models}
Two main models are compared:
\begin{itemize}
	\item \textbf{Linear Regression Baseline:} Standard least-squares regression with StandardScaler normalization.
          Target: establish a simple interpretable baseline.
	\item \textbf{Random Forest Regressor Ensemble:} 30 trees, max depth 10, random state 42.
		  Designed to capture non-linear fare dependencies, including distance-weather and location-time interactions.
\end{itemize}
No hyperparameter tuning was performed; parameters were chosen based on domain knowledge and
computational efficiency for this dataset.

\section{Model Comparison Summary}
The comparison reveals clear differences in model philosophy and capability.
Linear Regression provides a simple, interpretable baseline with explicit feature contributions.
Its prediction rule has a single global equation, so each feature coefficient has a direct and consistent meaning across all samples:
\begin{equation}
\hat{y}=w_0+\sum_j w_j x_j.
\end{equation}
This makes it easier to explain why a fare prediction increases or decreases when one input changes.
Random Forest captures complex non-linear patterns, delivering superior performance at the cost of interpretability.
For this application, the $R^2$ improvement from 0.233 to 0.939 and MAE reduction from \$6.58 to \$1.30
justify the shift toward the ensemble method.

\section{Pros and Cons by Model}
\textbf{Linear Regression (Baseline)}

\textbf{Pros:}
\begin{itemize}
	\item Simple and easy to interpret.
	\item Fast to train and evaluate.
	\item Useful as a transparent baseline for comparison.
\end{itemize}

\textbf{Cons:}
\begin{itemize}
	\item Lower predictive performance on this dataset (MAE = \$6.58, $R^2$ = 0.233).
	\item Cannot capture complex non-linear interactions as effectively.
\end{itemize}

\textbf{Random Forest (Ensemble)}

\textbf{Pros:}
\begin{itemize}
	\item Best predictive performance on this dataset (MAE = \$1.30, $R^2$ = 0.939).
	\item Captures non-linear interactions between features.
	\item Less sensitive to feature scaling.
\end{itemize}

\textbf{Cons:}
\begin{itemize}
	\item Harder to interpret than Linear Regression.
	\item Can be sensitive to data quality and sampling effects.
\end{itemize}

\section{Conclusion}
Random Forest regression achieves the best performance among tested models on this Uber/Lyft dataset, explaining 93.9\% of fare variance with a mean absolute error of \$1.30.
Despite the strength of the ensemble approach, a linear baseline is provided for transparency and comparison.
Future work should explore temporal validation to quantify drift sensitivity,
and integration of external data (e.g., real-time traffic conditions) to improve generalization.

\section{Threats to Validity}
The evaluation uses repeated random holdout splits rather than a strict temporal split, so deployment-time drift effects may still be underrepresented.
Confidence intervals are estimated from split-level empirical percentiles, which improves robustness to non-normal metric distributions but can still underestimate broader data-generation uncertainty.
The sampled subset (20,000 trips) may not capture all rare demand or weather regimes, and this can affect tail-error behavior.
Finally, because surge multiplier fields are excluded, the model estimates fare primarily from trip context and weather proxies rather than direct dynamic-pricing signals.
