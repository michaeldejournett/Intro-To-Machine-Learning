\begin{abstract}
This two-page report studies ride fare prediction for Uber and Lyft trips using supervised learning.
Given 14 trip-level features (distance, time, location, vehicle type, and weather conditions), the goal is to predict fare amount.
We evaluate baseline Linear Regression and ensemble Random Forest models, report test-set performance with 95\% confidence intervals, and compare model
quality using regression metrics and normalized confusion matrices.
The best model (Random Forest) achieves MAE = \textbf{\$1.23}, RMSE = \textbf{\$2.22}, and $R^2=\textbf{0.943}$
with a 95\% confidence interval for MAE of \textbf{[1.182, 1.268]}.
Results indicate that Random Forest provides the best trade-off between accuracy,
stability, and practical usability for this dataset, outperforming the linear baseline.
\end{abstract}

\section{Introduction}
Fare prediction is useful for pricing transparency, dispatch planning, and customer experience.
This project compares baseline (linear) and ensemble regression models on Uber/Lyft trip data to estimate fares.
Our final results are MAE = \textbf{\$1.23}, RMSE = \textbf{\$2.22}, and $R^2=\textbf{0.943}$,
with 95\% confidence intervals computed via bootstrap resampling (500 iterations).
Among the tested methods, Random Forest performs best and is selected as the final approach, consistent with prior findings that tree ensembles capture non-linear structure effectively \cite{breiman2001random,james2013isl,hastie2009elements}.
The report concludes with model strengths/limitations and practical implications for real-world deployment.

\section{Related Work}
Random Forests are widely used for tabular prediction tasks with heterogeneous features and non-linear interactions \cite{breiman2001random}.
Classical linear models remain useful as transparent baselines and diagnostic references \cite{james2013isl,hastie2009elements}.
Motivated by this literature, we report both a linear baseline and an ensemble model, and evaluate each with the same train/test split and metric suite.

\section{Problem Formulation}
Let $\mathbf{x} \in \mathbb{R}^{14}$ denote trip features and $y \in \mathbb{R}$ denote fare amount (USD).
The learning task is to estimate a function $f$ such that:
\begin{equation}
\hat{y} = f(\mathbf{x}), \quad \text{with objective } \min_f\; \mathbb{E}[\ell(y,\hat{y})].
\end{equation}
Inputs include distance (miles), temporal features (hour, day-of-week, month), location encodings (source, destination),
vehicle type (cab type, service name), and weather conditions (temperature, humidity, wind, pressure, rain, clouds).
Output is fare price. Data splits: 80\% train, 20\% test. Preprocessing included median imputation for weather and
Label Encoding for categorical features; no scaling was applied to Random Forest.

\section{Models}
Two main models are compared:
\begin{itemize}
	\item \textbf{Linear Regression (Baseline):} Standard least-squares regression with StandardScaler normalization.
          Target: establish a simple interpretable baseline.
	\item \textbf{Random Forest Regressor (Ensemble):} 30 trees, max depth 10, random state 42.
		  Designed to capture non-linear fare dependencies (e.g., distance-weather and location-time interactions).
\end{itemize}
No hyperparameter tuning was performed; parameters were chosen based on domain knowledge and
computational efficiency for this dataset.

\section{Model Comparison Summary}
The comparison reveals clear differences in model philosophy and capability.
Linear Regression provides a simple, interpretable baseline with explicit feature contributions.
Random Forest captures complex non-linear patterns, delivering superior performance at the cost of interpretability.
For this application, the $R^2$ improvement from 0.480 to 0.943 and MAE reduction from \$5.30 to \$1.23
justify the shift toward the ensemble method.

\section{Pros and Cons of the Approach}
\textbf{Pros:}
\begin{itemize}
	\item Strong predictive accuracy ($R^2=0.943$) and low mean absolute error (MAE=\$1.23).
	\item Robustness to nonlinear interactions and feature scaling invariance.
	\item Feature importance outputs enable understanding of dominant price drivers (service type, distance).
	\item Inference is fast for real-time applications.
\end{itemize}

\textbf{Cons:}
\begin{itemize}
	\item Limited extrapolation beyond training distribution; high-variance edge cases may be mispredicted.
	\item Sensitivity to data quality and missing-value imputation strategy.
	\item Tree ensemble interpretability lower than linear regression; individual decision paths not easily explained.
	\item Temporal patterns (time-of-day surge) captured indirectly via trees, not explicitly modeled.
\end{itemize}

\section{Conclusion}
Random Forest regression achieves the best performance among tested models on this Uber/Lyft dataset, explaining 94.3\% of fare variance with a mean absolute error of \$1.23.
Despite the strength of the ensemble approach, a linear baseline is provided for transparency and comparison.
Future work should explore repeated-split or temporal validation to quantify drift sensitivity,
and integration of external data (e.g., real-time traffic conditions) to improve generalization.

\section{Threats to Validity}
The evaluation uses a single random 80/20 split, so reported performance may vary under different partitions.
Confidence intervals are estimated via bootstrap resampling on test predictions; these intervals quantify predictive-sample variability and may underestimate full training uncertainty.
The sampled subset (20,000 trips) may not capture all rare demand or weather regimes, and this can affect tail-error behavior.
Finally, because surge multiplier fields are excluded, the model estimates fare primarily from trip context and weather proxies rather than direct dynamic-pricing signals.
